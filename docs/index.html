Architecture

Three layers of runtime defence

Built for AI agents in production. Every layer is independent, auditable, and operates without modifying model weights.

LAYER 01 — SCOUT

Threat Scoring

Evaluates every incoming request against a configurable threat matrix. Pattern-matched and scored 0–10 before any agent action executes.

threshold: 7.5 / 10.0
LAYER 02 — NITPICKER

Semantic Veto

Detects camouflaged prompt injection via cosine similarity against a signed blocked-pattern library. Catches rewording and social engineering that keyword filters miss.

similarity threshold: 0.45
LAYER 03 — JUDGE

Cryptographic Audit

Every decision produces an HMAC-signed receipt tied to the current Ed25519 policy manifest. Tamper with the log and the signature breaks.

algorithm: Ed25519 + HMAC-SHA256
Trust & Transparency

What it does.
What it doesn't.

This is a security tool. We'd rather you know the limits upfront than discover them in production.

Mechanism Guarantee
Ed25519 manifests Policy rules are cryptographically signed. Tampering breaks the signature.
HMAC receipts Every decision is logged with a signed receipt. Silent log modification is detectable.
Semantic veto Detects camouflaged injection via semantic similarity, not just keywords.
NIST AI RMF Controls mapped to GOVERN, MAP, MEASURE, and MANAGE functions.
181/181 tests Full test suite gates every release. No untested surface in core.

Current Limitations

  • ✗ Not SOC2 certified — planned Phase 3
  • ✗ Not horizontally scaled — single node; Redis multi-node on roadmap
  • ✗ Not a network firewall — operates at application layer only
  • ✗ Supply chain attacks stages 1–4 are out of scope — covers stages 5–6 (token replay, secret enumeration)
  • ✗ Not a replacement for model alignment — this is a runtime guardrail
