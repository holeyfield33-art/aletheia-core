# Support

Thanks for using Aletheia Core. Please use the right channel for the right
question — it gets you a faster answer and keeps the issue tracker focused on
bugs and feature work.

## Asking a question

- **Usage / "how do I…" questions:** open a thread under
  [GitHub Discussions](https://github.com/holeyfield33-art/aletheia-core/discussions).
  Please search existing threads first.
- **Documentation issues** (typo, broken link, unclear instruction): open a
  GitHub issue with the `docs` label.
- **Hosted demo problems** (`app.aletheia-core.com/demo`): see
  [`docs/LAUNCH_GUIDE.md` → "Hosted demo key persistence"](docs/LAUNCH_GUIDE.md).
  If the demo returns `demo_unavailable`, that is the operator runbook.

## Reporting a bug

Open a GitHub issue using the **bug report** template. Include:

- The exact steps to reproduce.
- The version (`package.json` `version` field for the frontend, or
  `git rev-parse HEAD` for the backend).
- The minimum command, request, or payload that triggers the bug.
- What you expected vs. what happened.

## Reporting a security vulnerability

**Do not open a public issue.** Follow the disclosure process in
[SECURITY.md](SECURITY.md). The fastest channel is **security@aletheia-core.com**.

## Commercial / enterprise support

For SLA-backed support, hosted plans, or compliance work, contact
**enterprise@aletheia-core.com**.

## Maintainer response expectations

This is an open-source project maintained by a small team. We aim to:

- Triage new issues within **5 business days**.
- Respond to security reports within **48 hours**.
- Land critical security fixes within **7 days** of confirmed reproduction.

These are best-effort targets, not guarantees. If you need guaranteed SLAs,
see the enterprise channel above.
